1. 아이콘 개선안
-아이콘은 확장 메뉴에 나오는 확장 기능을 스위치하거나, 설정, 커스텀 기능을 쓰게 해주는 버튼임.
-현재 호버-하이라이트의 범위가 아이콘의 그래픽 범위와 일치하는 듯함. 그래서 얇은 선이 있을 경우, 빈 공간에는 호버가 풀려버림 -따라서 아이콘의 호버 범위는 사각형으로 진행하며, 모든 아이콘의 호버 범위는 통일한다.
- 현재 아이콘은 직관적이지 않으며, 이쁘지도 않다. 그래서 개선이 필요해 개선안을 말해보고자 한다.
- 오른쪽 맨위 - 셀렉션(기본)/컴포지션 토글 아이콘 
'셀렉션'은 사각형에 꼭지점에 작은 사각형이 달린 형태로 한다./ '컴포지션'은 가로로 긴 직사각형이며, 정 중앙에 작은 +표시를 더한다. 셀렉션은 파란색 / 컴포지션은 주황색 아이콘으로 표시한다. 둘다 라인아이콘
- 오른쪽 중간 - 마스크 토글 아이콘 / 마스크 모양 (색이 칠해진 직사각형에 거윤데 동그라미 투명 부분) / 토글이 켜져있는게 기본이며 파란색으로 표시하고, 토글이 꺼져있으면 어두운 회색으로 꺼져있음을 표시한다.
- 오른쪽 아래 - 설정 톱니바퀴인데 안에가 채워져있고, 가운데 원형은 비어있는 형태이다. 마우스를 호버시키면 파란색이 되며, 선택되면 설정창이 떠야한다.

2. 설정 문제
- 현재 설정은 CEP로, 설정 버튼을 토글시키고, Extentsion - 해당 플러그인 을 켜야 설정창을 확인할 수 있다.
- 따라서 설정 아이콘은 -> extension의 해당플러그인 CEP를 여는 형태가 되어야한다.
- 그리고 사용자가 익스텐션에서 해당 플러그인 창을 연다고해도, 설정 화면이 나와야한다. 즉, 이 화면은 늘 설정 화면이어야함.
- 현재 커스텀 그리드는 설정의 저장과 연동이 되어있지 않음. 커스텀의 UI는 이렇게 되어야함
왼쪽에 세로로 3개의 커스텀 앵커 아이콘이 있고. 오른쪽에는 4x4셀-그리드를 가진 정사각형이 있어야함. 정사각형 위에 마우스 호버시, 마우스 모양이 바뀔 필요는 없음.
- 앵커 표시는 실제 에펙에서 앵커 인디케이터와 비슷하게 만들어줬으면함.
- 마우스 클릭시 해당 좌표가 나오는 것은 좋음, 그리고 드래그를 했을때도 옮길 수 있는 인터페이스가 추가되었으면 함. 그리고 드래그와 키의 연동도 같이 있었으면함.
shift + 드래그 - 앵커를 수직방향으로 옮김.
ctrl + 드래그 - 5x5의 그리드에 스냅 이동
- 좌표는 소수점이 아닌 정수%로 표시되었으면함. 물론 0아래나 100을 넘어선 숫자도 가능함.
- 그리고 해당 앵커를 좌우반전하거나 상하반전하는 키도 있으면 좋을듯.
- 저장은 일단 왼쪽커스텀 아이콘을 클릭(아이콘 푸른색 토글)하고 오른쪽 그리드 위에 앵커를 설정하면 자동으로 저장이 되는걸로 함.
- 디폴트는 3개다 중앙으로 되어있음.
- 그리드의 행렬 갯수를 정하는 UI는 사각형 안의 숫자보다는 파란색 숫자이고 호버하면 <-> 마우스 모양이 너비 조정하는 모양으로 변환, 그리고 좌우로 숫자를 조절할 수 있으며, 클릭할 경우 숫자도 직접입력 가능. 최소는 3이고, 최대는 7임, 최소 미만은 3으로 설정되며, 최대 초과는 7로 설정됨
- 모드는 좌 셀렉션 우 컴포지션 // 좌 마스크 적용 우 마스크 적용하지 않음 이런식으로 표현하고 설정되어 있는 것으로 컬러가 바뀌게끔. 셀렉션과 컴포지션 색상은 푸른색 주황색 동일하고, 마스크는 토글된 모드의 글씨에 푸른색으로 표시됨. 
- 각 모드에는 해당 아이콘이 있어야 구분이 잘될듯함. 단 꺼져있을 경우 어두운 회색으로 통일이고, 선택된 경우 셀렉션, 컴포지션은 각각의 컬러로, 마스크는 푸른색으로 표시되었음함

3. 단축키
- 현재 단축키는 변경이 불가능하지만, 단축키 설정을 넣었으면함.
- 그 전에 우선 토글 클릭 모드는 이제부터 Y~Y, Y를 빠르게 두번 누르는 걸로 호출되게끔 변경 
- 단축키 설정은 어떤걸 홀드할껀지, 어떤걸 빠르게 두번 누를지를 설정하는거임


4. 그 외
- 구현 안된 기능들을 구현해줬으면 함. 설정 맨 밑에 배너형식으로 내가 쓸 로고를 하나 넣었으면 함. 아직 이름도 안정했으니. 귀여운 그림 알아서 넣어주면 좋겠음
- 맨 위에도 배너 형식으로 플러그인의 이름이 들어가면 좋겠다는 생각, 하지만 아직 가제이니 적당히 넣어주면 될듯함.
- 현재 투명도 옵션이 전혀 이뤄지지 않고 있음. 그리고 팝업경고창이 뜨는 것보다는, 해당 UI의 아래쪽에 붉은 글씨로 경고가 뜨는것이 좋을듯함.
- 한국에 배포할거니, 언어 설정이 가능하면 하나 있으면 좋을듯함.
- 설정 맨위에 Y를 눌렀을때 나오는 그리드 창이 그대로 떴있으면 좋을듯함. 그리고 행렬을 바꿀때 프리뷰 역할을 하며, 실제로 클릭으로 작동도 할 수 있으면 좋을듯함. C++을 CEP위에 구현하는게 어렵다면, 프리뷰는 CEP로 새로운 UI로 표현해도 상관없음
- 그리고 그리드 크기 조절도 투명도 조절 위에 있으면함. 지금상태가 50이고 ---o--- 이런 형식으로 왼쪽 오른쪾을 이동하면서 크기 조절하면 좋을읏함. 총 5단계이며, 디폴트는 지금보다 -20%의 형태로 한다. 그리고 +20%당 한단계로 설정해서 -40, -20, 0 ,+20 , +40 
- 셀의 수가 많아져도, 그리드의 크기는 변하지 않음.
